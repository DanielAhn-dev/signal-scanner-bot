# 가상 자동매매 월~금 실전 운용 플레이북

- 기준일: 2026-04-19
- 대상: 가상 자동매매를 주중 일관되게 운용하려는 사용자
- 목적: 월요일 진입, 화~금 관리, 장전/장중/장후 실행 타이밍을 고정해 과매매와 규칙 흔들림을 줄인다.

---

## 1. 핵심 원칙

1. 월요일은 진입일, 화요일~금요일은 관리일로 역할을 분리한다.
2. 장중 자동사이클은 10분 창 기준으로만 반복하고, 같은 창 중복 실행은 허용하지 않는다.
3. 실행 전 테스트, 실행 후 점검을 한 세트로 고정한다.
4. 리스크 신호가 높을수록 신규 진입보다 보유 관리 우선으로 전환한다.

---

## 2. 왜 월요일 운용을 분리하는가

현재 자동사이클은 모드별로 역할이 다릅니다.

- `monday`: 신규 진입(매수 슬롯) 중심
- `daily`: 보유 포지션 점검(익절/손절) + 필요 시 제한적 재진입
- `auto`: 실행 시점 재판단 모드지만 운용상 `daily`와 동일한 관리 사이클로 보는 것이 안전

따라서 보유가 0인 상태에서 주간 시작 포지션을 만드는 동작은 `monday`를 기준으로 잡는 것이 일관적입니다.

---

## 3. 주간 표준 루틴

### 일요일 (준비)

1. `/투자금`으로 자금, 분할, 목표수익, 성향, 일손실 한도를 확인
2. `/브리핑`, `/시장`으로 장세 점검
3. `/자동사이클 점검 진입` 1회 실행
4. 테스트 결과에서 스킵/오류 사유를 확인하고 월요일 실행 계획 확정

### 월요일 (진입일)

1. 08:40~08:55: `/자동사이클 점검 진입`
2. 09:05~09:15: `/자동사이클 실행 진입` (실집행 1회)
3. 실행 후: `/보유` -> `/거래기록 7` -> `/관심대응` 순서로 점검

운용 메모:
- 월요일 진입 실집행은 1회만 권장
- 같은 날 반복 실행은 과다 진입 가능성이 커서 금지

### 화요일~금요일 (관리일)

1. 장전 또는 장초: `/자동사이클 점검`
2. 실집행: 평일 09:00~15:30 KST 장중 자동사이클이 10분 단위로 반복
3. 체결 알림 수신 시 버튼으로 `/보유`, `/보유대응`, `/거래기록 최근 7일`을 바로 확인
4. 장중/장후: `/보유`, `/보유대응`, `/자동매도점검`으로 보유 리스크 관리
5. 금요일 장후: `/리포트 월간` 또는 `/거래기록 30`으로 복기

---

## 4. 보유 상태별 실행 규칙

### A. 보유 0개

1. 주간 첫 진입은 `진입` 모드 기준
2. 주중(화~금) 보유 0개가 되면, 재진입이 필요할 때 `진입` 모드 점검 후 1회 실행
3. 보유가 0인데 기본 실행만 반복하기보다 필요 시 `진입` 점검을 먼저 본다

### B. 보유 1개 이상

1. 기본은 `daily` 또는 `auto` 장중 10분 자동운영
2. 목표는 신규 탐색보다 보유 익절/손절 규칙 준수
3. 리스크 신호가 높으면 `/전략선택`으로 방어 전략을 먼저 적용

---

## 5. 일일 실행 체크리스트

1. 오늘 실행 모드를 하나로 확정했는가 (`monday` 또는 `daily/auto`)
2. 테스트를 먼저 실행했는가
3. 장중 자동운영이 켜져 있다면 같은 10분 창 중복 실행이 없는가
4. 실행 후 `/보유`와 `/거래기록` 또는 체결 알림 버튼으로 결과를 확인했는가
5. 필요 시 `/전략선택`으로 방어 강도를 조정했는가

---

## 6. 금지 패턴

1. 월요일 `monday` 실집행을 같은 날 여러 번 반복
2. 테스트 없이 바로 실집행
3. 손절/익절 이벤트가 있었는데 장후 점검 없이 다음 날로 넘김
4. 리스크 HIGH 신호에서 신규 진입을 계속 늘림

---

## 7. 권장 파라미터 범위 (보수형 기준)

- 매수 슬롯: 1~2
- 최대 보유: 6~8
- 최소 점수: 72 이상
- 익절: 7~9%
- 손절: 3~4%

파라미터 변경 시에는 반드시 아래 순서를 지킵니다.

1. 테스트 실행
2. 장중 자동운영 또는 단일 실집행 중 하나만 선택
3. 장후 결과 복기

# 신용/공매도 데이터 수동 입력 가이드 (2026-05-12)

## 📌 현황

**KRX API 차단으로 자동 수집 불가능**  
→ 웹 UI에서 **수동으로 매주 입력**하는 방식으로 전환

---

## ✅ 기능

| 항목 | 상태 |
|------|------|
| 공매도 데이터 | 📝 수동 입력 |
| 신용비율 | 📝 수동 입력 |
| DB 저장 | ✅ 자동 (Supabase) |
| 차트 표시 | ✅ 자동 (최신 데이터) |

---

## 🎯 사용 방법

### 1️⃣ 웹 앱 열기
대시보드 또는 설정 페이지에서 "신용/공매도 데이터" 섹션 찾기

### 2️⃣ 데이터 입력
```
거래 날짜: 2026-05-10
공매도비율(%): 4.5
공매도잔고(주): 15000000

신용비율(%): 3.2
```

### 3️⃣ 저장
버튼 클릭 → Supabase에 자동 저장 → 차트/분석에 즉시 반영

---

## 📊 입력 예시

| 날짜 | 공매도비율 | 신용비율 | 비고 |
|------|----------|--------|------|
| 2026-05-10 | 4.5% | 3.2% | 금요일 |
| 2026-05-11 | 4.6% | 3.3% | 토요일 (휴장) |
| 2026-05-12 | 4.4% | 3.1% | 일요일 (휴장) |

**매주 월요일 오후에 지난주 금요일 데이터 입력**

---

## 🔧 기술 세부사항

### Supabase 테이블
- `stock_credit_short_daily`: 일별 데이터 저장
  - `code`: 종목 코드
  - `date`: 거래일
  - `short_ratio`: 공매도비율
  - `credit_ratio`: 신용비율

### API 엔드포인트 (개발자용)
```typescript
POST /api/credit-short/save
{
  "code": "005930",
  "date": "2026-05-10",
  "short_ratio": 4.5,
  "credit_ratio": 3.2
}
```

---

## 📝 참고

- **정확도**: 수동 입력이므로 100% 정확
- **주기**: 매주 1회 (금요일 마감 후)
- **소요시간**: 5분
- **자동화**: Naver/증권사 API 복구 시 언제든 가능

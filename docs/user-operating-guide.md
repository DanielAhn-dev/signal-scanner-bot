# Signal Scanner Bot 운영 가이드

- 기준일: 2026-04-18
- 대상: 텔레그램 봇을 실제 매일 운용하려는 사용자와 운영자
- 원칙: 기능을 많이 쓰는 것보다 같은 순서를 반복해서 쓰는 것이 중요합니다.

> 이 문서는 기능 사전이 아니라 운영 문서입니다. 무엇을 언제 보고, 어떤 명령으로 확인하고, 언제 멈춰야 하는지까지 한 번에 정리합니다.

---

## 빠른 시작

1. `/온보딩`으로 기본 흐름을 먼저 확인합니다.
2. `/투자금 300만원 3 8 안전형 5`처럼 자금, 분할 횟수, 목표 수익률, 성향, 일손실 한도를 먼저 고정합니다.
3. 장 시작 전에는 `/경제` → `/시장` → `/브리핑` 순서로 시장을 먼저 보고, 리스크 신호가 높으면 `/전략선택`으로 대응 전략을 고정합니다.
4. 후보를 찾을 때는 `/스캔 [섹터]` 또는 `/눌림목`으로 압축한 뒤 `/종목분석`, `/재무`, `/수급`으로 검증합니다.
5. 진입 전에는 `/종목분석 [종목]`과 `/투자금` 결과가 동시에 납득될 때만 들어갑니다.
6. 장 마감 후에는 `/관심`, `/보유`, `/프로필`, `/리포트 월간`으로 복기합니다.

## 1. 이 봇을 어떻게 써야 하는가

이 봇은 한 번에 모든 기능을 쓰라고 만든 도구가 아닙니다. 실제 운용은 아래 여섯 단계로 나누어 반복하는 것이 맞습니다.

1. 시장 파악: 오늘 무리해도 되는 날인지 먼저 확인합니다.
2. 후보 탐색: 진입 후보를 3개에서 5개로 줄입니다.
3. 정량 검증: 점수, 재무, 수급으로 후보를 걸러냅니다.
4. 실행 판단: 진입가, 손절선, 목표가, 포지션 크기를 함께 봅니다.
5. 보유 관리: 보유 종목과 관심 종목을 같은 기준으로 점검합니다.
6. 복기와 자동화: 리포트와 자동화 결과로 다음 행동을 조정합니다.

이 순서를 지키면 기능이 많아도 실제로는 하루에 쓰는 명령이 제한됩니다. 반대로 이 순서를 건너뛰면 명령이 많을수록 판단이 흔들립니다.

## 2. 매일 보는 권장 순서

### 장 시작 전 5분

1. `/경제`
2. `/시장`
3. `/브리핑` 또는 `/다음섹터`
4. 리스크 신호가 MEDIUM 이상이면 `/전략선택`

이 단계의 목적은 매수 후보를 찾는 것이 아니라 오늘 리스크를 먼저 판단하는 것입니다. 변동성, 환율, 금리, 시장 국면이 불리하면 후보가 보여도 크기를 줄이거나 쉬는 것이 맞습니다.

### 후보 압축 10분

1. `/섹터`
2. `/스캔 [섹터명]`
3. `/눌림목`

여기서는 진입 후보를 3개에서 5개 수준으로만 추립니다. 많이 찾는 것이 목표가 아닙니다.

### 진입 전 검증

1. `/종목분석 [종목명 또는 코드]`
2. `/재무 [종목명 또는 코드]`
3. `/수급 [종목명 또는 코드]`
4. `/뉴스 [종목명 또는 코드]`

이 네 가지 중 둘 이상이 어색하면 진입을 보류합니다. 특히 `/종목분석`에서 제시하는 진입 구간과 손절선이 애매하면 억지로 들어가지 않습니다.

### 장중 또는 장마감 후 관리

1. `/관심`
2. `/관심대응`
3. `/보유`
4. `/보유대응`
5. `/자동매도점검`

이 단계는 이미 가진 포지션을 정리하는 시간입니다. 새 종목을 찾는 흐름과 섞지 않는 것이 좋습니다.

## 3. 핵심 명령 체계

### 시작과 설정

- `/온보딩` : 처음 사용자용 기본 루틴 안내
- `/도움말` : 전체 명령 요약
- `/투자금 [금액] [분할횟수] [목표수익률] [성향] [일손실한도%]` : 투자 계획 기준 고정
- `/프로필` : 내 사용 습관, 규칙 위반, 활동 통계 확인

### 시장 파악

- `/경제` : 거시 지표와 위험선호 확인
- `/시장` : 종합 시장 진단
- `/브리핑` : 장전 핵심 요약 + 리스크 신호(LOW/MEDIUM/HIGH)
- `/전략선택` : 위험 대응 전략 선택(HOLD_SAFE, REDUCE_TIGHT, WAIT_AND_DIP_BUY)
- `/섹터` : 주도 섹터 랭킹
- `/다음섹터` : 자금이 유입되는 다음 후보 섹터
- `/알림` : 이상징후 점검

### 후보 탐색

- `/스캔 [섹터]` : 섹터 기반 눌림목 스캐너
- `/눌림목` : 눌림목 후보 탐색
- `/종목 [섹터]` : 섹터 대표 종목 탐색
- `/kospi`, `/kosdaq` : 보수형 추천 TOP 5
- `/etf`, `/etf 추천`, `/etfcore`, `/etftheme` : ETF 허브와 추천

### 정량 검증

- `/종목분석 [종목]` : 점수, 진입가, 손절선, 목표가, 포지션 크기 확인
- `/재무 [종목]` : 핵심 재무 지표 확인
- `/수급 [종목]` : 외국인, 기관 흐름 확인
- `/뉴스 [종목]` : 관련 뉴스 확인
- `/etf 정보 [ETF]` : NAV, 괴리율 확인
- `/etf 분배금 [ETF]` : 분배금, 배당락 일정 확인

### 추적과 기록

- `/관심` : 관심 종목 추적
- `/관심추가 [종목]`, `/관심제거 [종목]`, `/관심초기화 확인`
- `/관심대응` : 관심 종목 대응 플랜
- `/보유` : 가상 보유 포트폴리오
- `/보유대응` : 익일 대응 플랜
- `/거래기록 [일수]` : 가상 매매 기록
- `/리포트` : 가능한 리포트 종류 안내
- `/리포트 월간` : 월간 성과 요약 텍스트
- `/리포트 주간`, `/리포트 포트폴리오`, `/리포트 거시`, `/리포트 수급`, `/리포트 섹터` : PDF 리포트

### 자동화와 운영

- `/자동사이클 점검` : 오늘 기준 자동매매 판단을 시뮬레이션
- `/자동사이클 실행` : 오늘 기준 자동매매를 실제 가상 반영
- `/자동사이클 점검 진입` 또는 `/자동사이클 실행 진입` : 신규 진입 판단을 강제로 확인
- `/자동매도점검` : 기준 충족 시 자동 매도 기록

자동사이클 최신 동작 요약:
1. 진입은 `눌림목 + 매집 포착` 하이브리드 우선(조건 통과분)
2. 월간 페이싱(연 10% 기준) 하회 시 기준 완화, 과열/손실 구간은 방어 복귀
3. 일손실 한도 도달 시 신규 매수 중단
4. 기본 결과 메시지는 3줄 요약(결과/상위사유/다음액션)

### 시나리오 A. 초보자 기본 루틴

1. `/경제`와 `/시장`으로 오늘 시장을 먼저 확인합니다.
2. `/브리핑`으로 후보 섹터를 확인합니다.

4. `/스캔 반도체`처럼 섹터를 지정해 후보를 줄입니다.
5. `/종목분석 삼성전자`, `/재무 삼성전자`, `/수급 삼성전자`로 검증합니다.
6. `/투자금` 기준과 `/종목분석` 결과가 같이 맞으면 진입합니다.
7. 장 마감 후 `/관심`, `/보유`, `/프로필`로 복기합니다.

### 시나리오 B. 이미 보유 종목이 있는 경우

1. `/보유`
2. `/보유대응`

보유 종목이 많은 날은 새 후보 탐색보다 기존 포지션 정리 우선입니다.

### 시나리오 C. 주간 점검

1. `/리포트 주간`
2. `/리포트 섹터`
3. `/리포트 수급`
4. `/리포트 월간`

주간 흐름을 볼 때는 종목 하나보다 시장, 섹터, 수급, 포트폴리오 순서로 보는 편이 안정적입니다.

## 5. 리포트를 어떻게 읽어야 하는가

### `/리포트 주간`

시장과 포트폴리오를 함께 보는 종합 리포트입니다. 이번 주 기본 문서로 가장 먼저 보는 리포트입니다.

### `/리포트 포트폴리오`

보유 종목, 수익률, 거래 기록을 중심으로 봅니다. 내가 실제로 무엇을 잘못하고 있는지 확인할 때 가장 직접적입니다.

### `/리포트 거시`

금리, 환율, 변동성처럼 종목 외부의 환경을 점검합니다. 종목 문제가 아니라 환경 문제일 수 있는 주간에 유용합니다.

### `/리포트 수급`

자금 흐름과 회전 방향을 파악합니다. 섹터 전환이 시작되는지 볼 때 먼저 확인합니다.

### `/리포트 섹터`

어떤 섹터가 강하고 약한지 압축해서 봅니다. 스캔 전 섹터 방향을 먼저 확인할 때 적합합니다.

### `/리포트 월간`

텍스트 기반 월간 성과 요약입니다. 승률보다 손익비, 규칙 위반, 최대 손실 같은 항목을 우선해서 읽는 것이 맞습니다.

### 브리핑 리스크 신호와 전략 반영 해석

`/브리핑`에는 리스크 신호 섹션이 표시됩니다.

- `LOW` : 기존 자동매매 규칙 유지
- `MEDIUM` : `/전략선택`으로 방어 강도를 높일지 결정
- `HIGH` : 신규 매수 억제 전략 우선 고려

전략별 자동매매 반영은 아래처럼 동작합니다.

- `HOLD_SAFE` : 보수 운용, 총 2종목까지 제한 진입하고 기존 포지션은 방어적으로 관리
- `REDUCE_TIGHT` : 손절 2%, 익절 4%로 강화
- `WAIT_AND_DIP_BUY` : 신규 매수 중단, 현금 비중 유지

## 6. 자동화 운영 가이드

자동화는 사람 판단을 대체하는 기능이 아니라, 같은 기준을 반복 집행하는 기능입니다. 그래서 실행 전에 dry-run이 우선입니다.

### 텔레그램에서 바로 점검

```bash
/자동사이클 점검
/자동사이클 실행
/자동사이클 실행 진입
```

### 로컬 또는 운영 환경에서 점검

```bash
pnpm autotrade:dry-run -- --mode=auto --maxUsers=50
pnpm autotrade:run -- --mode=auto --maxUsers=20
pnpm autotrade:enable -- --all-active --enable=true
pnpm autotrade:enable -- --tgIds=123456789 --enable=false
```

### 크론 수동 호출 예시

```bash
curl -H "Authorization: Bearer $CRON_SECRET" "https://signal-scanner-bot.vercel.app/api/cron/virtualAutoTrade?mode=auto&dryRun=true&maxUsers=20"
```

### 자동화에서 꼭 지킬 규칙

1. 새 파라미터를 적용하기 전에는 항상 dry-run으로 먼저 확인합니다.
2. 자동화 결과는 `/보유`, `/거래기록`, `/리포트 포트폴리오`로 다시 점검합니다.
3. 자동화를 켰더라도 `/경제`, `/시장` 단계는 사람이 계속 봐야 합니다.

### 실전 운용 요약 (월~금)

1. 주간 첫 진입 전에는 `/자동사이클 점검 진입`으로 점검만 하고, 실집행은 필요할 때 `/자동사이클 실행 진입` 1회로 제한합니다.
2. 일반 운용은 `/자동사이클 점검` 후 `/자동사이클 실행` 흐름으로 하루 1회만 고정 실행합니다.
3. 보유가 0개가 된 뒤 재진입이 필요하면 `/자동사이클 점검 진입`으로 먼저 확인합니다.
4. 시간대별 체크리스트와 예외 대응은 `docs/virtual-autotrade-weekly-playbook.md`를 기준으로 운용합니다.

## 7. 운영자가 알아야 할 문서 갱신 규칙

이 문서는 기능이 바뀔 때 같이 수정해야 하는 기준 문서입니다. 아래 상황에서는 코드 변경과 함께 이 문서를 갱신해야 합니다.

1. 사용자 명령이 추가되거나 제거된 경우
2. 명령 이름, 별칭, 도움말 문구가 바뀐 경우
3. 리포트 종류나 PDF 전달 방식이 바뀐 경우
4. 자동화 실행 방식이나 파라미터가 바뀐 경우
5. 초보자 권장 사용 흐름이 바뀐 경우

운영자가 점검할 파일은 아래 순서로 고정하는 것이 좋습니다.

1. `src/bot/commandCatalog.ts`
2. `docs/user-operating-guide.md`
3. `README.md`
4. 필요 시 `docs/onboarding-roadmap.md` 또는 `docs/virtual-autotrade-ops.md`

PDF는 원본이 아니라 생성물입니다. 문서 수정 후에는 아래 명령으로 PDF를 다시 생성합니다.

```bash
pnpm docs:guide:pdf
```

## 8. 자주 하는 실수

### 기능을 다 쓰려고 하는 경우

이 봇은 모든 기능을 다 쓰는 사람보다, 같은 흐름을 반복하는 사람이 더 잘 씁니다. 매일 쓰는 핵심 명령은 `/경제`, `/시장`, `/브리핑`, `/스캔`, `/종목분석`, `/보유`, `/리포트` 정도면 충분합니다.

### 스캔 결과를 바로 매수하는 경우

`/스캔`은 후보 탐색 단계입니다. 반드시 `/종목분석`, `/재무`, `/수급`으로 검증을 거쳐야 합니다.

### 손절 기준보다 뉴스 한 줄에 흔들리는 경우

뉴스는 보조 정보입니다. 기본은 미리 잡아 둔 진입가, 손절선, 포지션 크기입니다.

### 자동화를 켜고 사람이 시장을 안 보는 경우

자동화는 규칙 집행 도구이지 책임 회피 도구가 아닙니다. 특히 고변동 장에서는 사람이 `/경제`와 `/시장`을 계속 봐야 합니다.

## 9. 운영 체크리스트

### 사용자 체크리스트

1. 오늘 `/경제`, `/시장`, `/브리핑`을 확인했는가
2. 리스크 신호가 MEDIUM 이상일 때 `/전략선택`을 먼저 적용했는가
3. 후보를 5개 이하로 줄였는가
4. 진입 전 `/종목분석`, `/재무`, `/수급`을 확인했는가
5. `/투자금` 기준을 벗어나는 진입을 하지 않았는가
6. 장 마감 후 `/보유` 또는 `/관심`으로 복기했는가

### 운영자 체크리스트

1. 명령 목록이 `src/bot/commandCatalog.ts`와 일치하는가
2. README 링크가 최신인가
3. PDF가 한글 깨짐 없이 생성되는가
4. 새 기능 추가 시 이 문서와 관련 운영 문서를 같이 갱신했는가

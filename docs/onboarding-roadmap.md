# 온보딩 · 기능개선 로드맵

이 문서는 초보자가 감정매매를 줄이고, 규칙기반으로 손실을 줄이는 데 초점을 둡니다.

> 중요: 빠른 자산 증식을 보장하는 방법은 없습니다. 목표는 **대형 손실 회피 + 기대값 관리 + 복리 가능한 생존율**입니다.

---

## 1) 초보자 권장 조회 순서 (매일)

### A. 장 시작 전 (5~10분)

1. `/경제` → 거시 리스크(변동성/환율/금리) 확인
2. `/시장` → 시장 국면(강세/중립/약세) 진단
3. `/브리핑` 또는 `/다음섹터` → 오늘 순환매 후보 파악

### B. 후보 압축 (10~20분)

4. `/스캔 [섹터]` + `/눌림목` → 진입 후보 3~5개 추림
5. 각 후보에 `/점수`, `/재무`, `/수급` 교차검증

### C. 실행 전 체크 (진입 직전)

6. `/매수 종목명`으로 진입/손절/목표 레벨 확인
7. `/투자금` 기준으로 분할매수 수량/리스크 확인
8. 조건 불충족 시 진입 보류 (무포지션 허용)

### D. 사후 관리 (장 마감 후)

9. `/관심`으로 보유/관심 종목 점검
10. `/프로필`로 사용 습관 확인, 규칙 위반 기록

---

## 2) 초보자 최소 규칙 (이것만은 고정)

- 1회 거래 손실 한도 고정 (계좌 대비 일정 비율)
- 손절 기준 사전 입력 후 변경 금지
- 분할 진입(기본 3회), 단일 종목 과집중 금지
- 하루 최대 손실/연속 손실 기준 도달 시 거래 종료
- 변동성 급등 구간에서는 포지션 축소

---

## 3) 현재 기능으로 가능한 시스템 매매 흐름

- 시장 필터: `/경제`, `/시장`
- 아이디어 생성: `/브리핑`, `/섹터`, `/다음섹터`
- 후보 선별: `/스캔`, `/눌림목`
- 정량 검증: `/점수`, `/재무`, `/수급`
- 실행/리스크: `/매수`, `/투자금`
- 기록/복기: `/관심`, `/프로필`, `/랭킹`, `/피드`

---

## 4) 단계적 개선안 (없으면 추가)

### Phase 1 — 온보딩 고정화 (단기)

- [완료] `/온보딩` 명령 추가
- [제안] `/온보딩 체크` (오늘 체크리스트 완료율)
- [제안] 초보자 모드 프로필 플래그 (`beginner=true`) 도입

### Phase 2 — 이상징후 감지 (중기)

- [완료] `/알림` 요청형 점검 (VIX/환율/금리/지수 급변 + 섹터 수급 급변)
- [진행] 섹터 급회전 정교화 (임계치/점수 동적 튜닝)
- [제안] 종목 이상거래 알림 (거래대금/변동률 급증)

### Phase 3 — 규칙집행 자동화 (중기)

- [제안] 진입 전 자동 점검 스코어(필수조건 충족률)
- [제안] 일손실 한도 도달 시 알림/거래중지 권고
- [제안] 손절 미이행 감지 알림

### Phase 4 — 전략 고도화 (중장기)

- [제안] 전략별 백테스트 리포트 (승률, 손익비, MDD)
- [제안] 시장국면별 가중치 자동 조정 (기술/재무/수급)
- [제안] 사용자별 성과 기반 추천 워크플로우

---

## 5) 운영팀 실행 순서

1. `/onboarding` 텔레그램 명령어 갱신
2. 초보자 모드 기준값 확정 (`/투자금` 기본값 정책)
3. 이상징후 알림 임계치 테이블 정의
4. 2주 단위로 사용자 로그 분석 후 규칙 튜닝

---

## 6) 기대효과 (현실적)

- 충동 진입 감소
- 손절 지연 감소
- 거래 일관성 증가
- 장기적으로 손실 변동성 축소

수익률 자체보다 먼저, **큰 손실을 피하는 시스템**을 만드는 것이 복리의 출발점입니다.

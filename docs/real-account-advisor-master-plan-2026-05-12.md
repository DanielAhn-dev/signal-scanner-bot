# 실계좌 폴더형 포트폴리오 어드바이저 마스터 플랜 (2026-05-12)

## 1) Decision Summary
- 목표: 가상매매 검증 시스템을 실계좌 의사결정 보조 시스템으로 확장.
- 핵심: 계좌(브로커/계좌명)별 보유 관리 + 종목별 액션(전체익절/부분익절/손절/추매) + 거시지표(금리/유가/CPI) 반영.
- 전략: 기존 virtual_positions/점수엔진을 재사용해 단계적으로 롤아웃.

## 2) Finance Specialist View
- 기대효과
  - 사용자 의사결정 일관성 향상: 감정 개입 감소, 규칙기반 대응 강화.
  - 계좌별 리스크 통제: ISA/연금/일반 계좌별 운용 원칙 분리 가능.
- 리스크
  - 과최적화 위험: 단기 신호 과신으로 불필요한 회전율 증가 가능.
  - 거시지표 오류 위험: CPI 최신성/출처 신뢰도 미흡 시 역방향 시그널 가능.
- 통제안
  - 액션은 항상 제안(Advisor)로 제공, 자동 실행 금지.
  - 거시 리스크는 가중치(임계치 조정)만 적용, 강제 청산 금지.
  - 로그 기반 사후평가: 제안 대비 실제 결과 추적.

## 3) Design Specialist View
- 사용자 여정
  - 계좌 폴더 생성/지정 -> 보유종목 입력 -> 종목카드에서 액션 제안 확인 -> 실제 주문 참고.
- UX 원칙
  - 한 화면에서 판단 완료: 종목상태 + 거시상태 + 계좌맥락 동시 노출.
  - 설명 가능성: 왜 이 액션인지(점수/경고/거시)를 카드에서 즉시 확인.
- UI 우선순위
  - 1차: 계좌 필터/표시, 거시 가중치 요약 배너.
  - 2차: 계좌별 요약(손익/현금/위험도), 액션 시뮬레이터.

## 4) Development Specialist View
- 아키텍처
  - DB: virtual_positions에 broker_name/account_name 추가.
  - API: positions/positions-maintenance에 계좌 메타 읽기/쓰기 확장.
  - Web: portfolio 화면에 계좌 폴더 필터 + 거시 가중치 반영.
- 단계
  - Phase 1(완료): 계좌 메타 저장/조회, 계좌별 필터, 거시 가중치(금리/유가/CPI) 반영.
  - Phase 2: 계좌별 리스크 예산(손실 한도, 종목수, 섹터캡), 계좌 단위 추천.
  - Phase 3: 제안-성과 피드백 루프(정확도 리포트) 및 전략 자동 튜닝.
- 운영
  - CPI는 우선 환경변수(MARKET_CPI_YOY)로 제공하고, 이후 공식 API 연동으로 전환.

## 5) Trade-off Matrix
- 높은 가치/낮은 복잡도: 계좌 메타 필드 추가, 화면 필터.
- 높은 가치/중간 복잡도: 거시 가중치 임계치 조정.
- 매우 높은 가치/높은 복잡도: 계좌별 리스크 엔진 + 백테스트 기반 파라미터 튜닝.

## 6) Final Recommendation
- 추천 경로: 단계적 확장.
  - 지금: 계좌 폴더 + 거시 반영 어드바이저를 실사용 시작.
  - 다음: 계좌별 리스크 정책/제한 추가.
  - 이후: 추천 정확도 추적/학습으로 신뢰도 고도화.

## 수익 극대화 우선순위 제안
1. 자동화 + 휴리스틱 병행: 규칙 기반 자동화와 휴먼 인터벤션 병행

3. 전략 포트폴리오화: 상관 낮은 여러 전략 운영
4. 실행 우선순위 최적화: 체결확률 높은 신호 우선집행
5. 모니터링 투자: 작은 이상도 즉시 감지

## 인터랙티브 실행 절차 (텔레그램 버튼 및 명령)
아래는 텔레그램 알림 메시지와 인라인 버튼에 매핑할 권장 명령(또는 callback)을 정리한 항목입니다. 실제 스크립트/엔드포인트 이름은 프로젝트에 맞게 바꿔 사용하세요.

- 알림: "시스템 가동 완료 — 시작 준비됨"
	- 텔레그램 버튼 레이블: **헬스체크 실행**
	- 텍스트 커맨드: `/healthcheck`
	- 서버 처리: `/api/control/healthcheck` 호출 → 내부적으로 `pnpm ts-node scripts/diagnose_scan_data.ts` 또는 `python scripts/update_market_data.py` 실행

- 알림: "오늘 계획 준비됨"
	- 텔레그램 버튼 레이블: **오늘 계획 보기**
	- 텍스트 커맨드: `/view_plan` 또는 버튼 클릭
	- 서버 처리: `/api/control/view_plan` → PDF 생성 후 파일 링크 전송 (`node scripts/md-to-pdf.js docs/시나리오-리포트.md docs/시나리오-리포트.pdf`)

- 알림: "자동매매 대기중"
	- 텔레그램 버튼 레이블: **자동매매 시작/재개**
	- 텍스트 커맨드: `/start_autocycle`
	- 서버 처리: `/api/control/start` → `pnpm run start:worker` 또는 `node build/api/worker.js` 실행

- 알림: "이상 징후 발생"
	- 텔레그램 버튼 레이블: **오토사이클 일시정지**
	- 텍스트 커맨드: `/pause_autocycle`
	- 서버 처리: `/api/control/pause` → 자동매매 루틴(오토사이클)을 안전하게 중지

- 알림: "긴급: 즉시 중지 권고"
	- 텔레그램 버튼 레이블: **전체 중지(비상)**
	- 텍스트 커맨드: `/stop_all`
	- 서버 처리: `/api/control/stop` → 워커·오더 프로세스 완전 중지 (`pnpm run stop:worker` 등)

추가 명령:
- `/generate_report` → 장마감후 리포트 생성(백테스트/요약) 호출

예시: 텔레그램 인라인 키보드 JSON (Bot API용)

```json
[
	[{"text":"헬스체크 실행","callback_data":"healthcheck"},{"text":"오늘 계획 보기","callback_data":"view_plan"}],
	[{"text":"자동매매 시작/재개","callback_data":"start_autocycle"},{"text":"오토사이클 일시정지","callback_data":"pause_autocycle"}],
	[{"text":"전체 중지(비상)","callback_data":"stop_all"}]
]
```

서버 측 구현 팁:
- Bot에서 `callback_data` 수신 시 내부 API(`/api/control/<action>`)를 호출하도록 라우팅하세요.
- 각 API는 인증(운영자 검증)과 안전 확인(2차 확인 또는 확인 토글)을 거쳐 명령을 실행하도록 구현하세요.
- 버튼을 대시보드에도 동일하게 노출하면 PC/모바일 모두에서 동일한 워크플로를 따를 수 있습니다.

팁: 텔레그램 명령과 버튼은 모두 지원하면 사용성이 좋습니다 — 텍스트 커맨드로 직접 입력하거나 버튼 클릭으로 빠르게 동작시키세요.
